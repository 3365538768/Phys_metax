数据集生成（auto_simulation_runner + modified_simulation）扩展任务

目标：生成更全面的仿真数据集，在「可训练/可评估」标签之外，显式记录渲染与视角相关配置，并以 2D 形式输出应力与形变轨迹（避免大体积三维场文件）。

一、train_config.json 中建议综合记录的字段（与 auto_simulation_runner 读取一致）

1) 已有/常用
   - num_simulations, sim_types, seed, output_root
   - base_config_by_sim_type
   - num_views：渲染方位角均匀分布的视角数量（与原先一致）
   - render_img, compile_multiview_videos（推荐，统一 images 与 stress_gaussian 的 mp4）或 compile_video（未写前者时）, delete_png_sequences_after_compile_video（可选：ffmpeg 成功后删对应 PNG 序列目录）, white_bg, debug
   - quiet（可选，默认 false）：为 true 时子进程加 `--quiet`（关闭仿真帧 tqdm、压低 Warp/Taichi 启动输出）。`auto_simulation_runner` 仍用 tqdm 显示批量 job 总进度与 ETA。命令行 `--quiet` 与配置二选一即可。
   - dataset_name（可选，默认不写或空）：非空时样本写入 `output_root/<dataset_name>/[dataset_split]/000000/`（6 位编号，与 `dataset_400/train/000000` 同级结构一致）。同批 `num_simulations` 按 job 顺序从 0 递增。仍写 `gt_parameters.json`；另写 **`gt.json`**：`object`/`action`/`params`/`regression` 与 `transform_dataset` 产物一致，并含 `sample_index`、`job_index`、`train_seed`、`ply_path`、`dataset_name`、`dataset_split` 等便于后续划分 train/test 或统一训练。
   - dataset_split（可选，默认 `train`）：命名数据集下的中间目录名；设为空字符串 `""` 则路径为 `output_root/<dataset_name>/000000/`。命令行可 `--dataset_name my_ds --dataset_split train` 覆盖 JSON。
   - 多卡（`--num_gpus`≥2）：**动态占卡**——`--num_gpus N` 为「最多同时 N 个仿真」，候选池为全部可见 GPU（非仅用 0..N-1），优先选剩余显存大且满足 `idle_gpu_*` 条件的卡；细节见 `idle_gpu_min_free_mib`、`idle_gpu_max_util_pct`、`idle_gpu_relax_util_when_free_mib` 等。**默认**对子进程自动加 `--quiet`，避免多条「仿真帧」tqdm 与 Warp/Taichi 横幅打乱总进度；调试多卡子进程时设 `verbose_subprocess_multi_gpu`: true。
   - output_deformation, output_stress, field_output_interval（体积 npz；可与下面「关闭体积」配合）
   - output_bc_info, output_force_info

2) 新增（渲染时间采样与视角）
   - num_render_views（可选）：显式声明「渲染视角数量」。若不写则与 num_views 相同；写 -1 亦表示沿用 num_views。
   - render_outputs_per_sim_second（可选，记为 N，默认 0）：**>0 时优先**。仿真总时长 T≈frame_num×frame_dt（秒），目标输出张数 K≈round(T×N)，在 [0,frame_num-1] 上含端点均匀取 K 个仿真步；RGB、应力 3DGS、flow、tracks_gaussian、应力热力图、下采样世界轨迹等与该时间轴一致。compile 多视角视频时 ffmpeg -framerate 也用 N，使视频时长≈T。≤0 时用 num_render_timesteps。
   - num_render_timesteps（可选，默认 0；仅当 render_outputs_per_sim_second≤0 时作为主开关）：
       * 0 或 ≥ 仿真总帧数：每个仿真时间步都输出一帧渲染/视角辅助（与原先每帧出图一致，文件名按输出序号 0000.. 对齐）。
       * 正整数 K：在整段仿真时间上**含端点均匀**取 K 个仿真帧做输出；RGB、应力热力图、2D 轨迹在该时刻同时采样，帧数一致。
     「总共渲染多少张图」在固定视角数 V 下，为 K×V（每个视角 K 张）；K 由 render_outputs_per_sim_second（N 模式）或 num_render_timesteps 控制。

3) 视角内 2D 产物（不写三维 stress/deformation 体数据时可关体积开关）
   - output_view_stress_heatmap：true 时，每个视角目录下写 stress_heatmaps/<view_name>/{out_idx:04d}.png
       基于 Cauchy 应力的 von Mises（与 save_stress_field 同式：σ=τ/det(F_trial)）。在**每个渲染采样帧**会先调用
       `recompute_particle_stress_from_F_trial(substep_dt)`，使 Kirchhoff τ 与 g2p 更新后的 F_trial 一致，再 splat。
       仅对光栅 radii>0 的粒子绘制（与 RGB 同一相机、同一 out_idx）。
   - output_view_flow_gaussian：true 时写 flow_gaussian/<view_name>/{out_idx:04d}.png。与主 RGB 同相机；第二遍 CUDA 3DGS（黑底），对随机下采样的 MPM 高斯用 colors_precomp 表示相邻输出帧像平面 (Δu,Δv) 的方向与幅值（Middlebury 风格 HSV→RGB），opacity≈原不透明度×幅值×opacity^α·(1/(dist+ε))^γ；非下采样粒子透明。不写 npz。flow_gaussian_vis_max_motion≤0 时按子集分位数定幅值饱和。
   - output_view_tracks_gaussian：true 时写 tracks_gaussian/<view_name>/{out_idx}.png（3DGS、黑底）：整物体 MPM 下采样，默认 tracks_gaussian_accum_mode=max 逐像素取大（线清晰）；add 为逐帧相加易糊；另可调 sigma_scale、point_opacity、accum_frame_weight、accum_decay、accum_midpoint。
   - output_subsampled_world_tracks：true 时写 tracks_subsampled_world/tracks_world.npz（下采样粒子世界坐标，无光栅化；时间轴与 num_render_timesteps 一致）；compile 时另可有 videos/tracks_subsampled_world_ortho_*.mp4（正交折线预览）。
   - output_view_stress_gaussian：true 时写 stress_gaussian/<view_name>/{out_idx}.png（与 RGB 同管线，预计算颜色+opacity）。
   - no_volumetric_stress_deformation：true 时，不写 deformation_field/、stress_field/ 下按帧大 npz（仍保留 meta、渲染与上述 2D 产物）。

二、单次仿真输出目录约定（在各自 job 的 output_path 下）

   images/<view_name>/{out_idx:04d}.png          # render_img 时
   stress_heatmaps/<view_name>/{out_idx:04d}.png # output_view_stress_heatmap
   stress_gaussian/<view_name>/{out_idx:04d}.png # output_view_stress_gaussian
   flow_gaussian/<view_name>/{out_idx:04d}.png      # output_view_flow_gaussian（稠密 flow 伪彩）
   meta/run_parameters.json                     # 汇总：仿真类型、帧数、渲染索引、材质与时间参数、各开关等
   videos/<view_name>.mp4                       # compile_multiview_videos（或 compile_video）且 render_img 时
   videos/<view_name>_stress_gaussian.mp4        # 同上且 output_view_stress_gaussian 时
   tracks_gaussian/<view_name>/{out_idx:04d}.png # output_view_tracks_gaussian
   videos/<view_name>_tracks_gaussian.mp4        # compile 同上且 output_view_tracks_gaussian 时
   videos/<view_name>_flow_gaussian.mp4          # compile + output_view_flow_gaussian
   tracks_subsampled_world/tracks_world.npz      # output_subsampled_world_tracks（下采样粒子世界坐标，无光栅化）
   videos/tracks_subsampled_world_ortho_<xy|xz|yz>.mp4  # compile 同上且 output_subsampled_world_tracks（正交折线预览）

三、与 my_model 训练数据的关系

   新字段主要影响仿真侧目录与 meta；构建 dataset_400 类布局时，应读取 meta/run_parameters.json 将 num_render_views_effective、num_render_outputs、render_frame_indices 等写入样本级 manifest，便于与视频帧对齐。

四、仿真原始输出目录（auto_simulation_runner）

   - 默认 `train_config` 中 `output_layout` 为 **`by_model`**：
     `auto_output/<PLY 文件名不含扩展名>/<obj__params__sim_type>/<obj>/...`
     即**先按物体**再按单次 run，不再使用 `auto_output/<bend|drop|...>/` 顶层。
   - 若需恢复旧结构，在 train_config 中设 `"output_layout": "by_action"`。

五、整理为「按 model 分子目录」的数据集（dataset_full_test 等）

   使用仓库根目录 `transform_dataset.py`，与 dataset_400 **同一套单样本内容**（images/、gt.json、boundary_conditions.json），仅顶层目录不同：

   ```bash
   python transform_dataset.py --layout by_model --out_name dataset_full_test [--copy_aux]
   ```
   （`transform_dataset` 已同时扫描「按动作」与「按物体」两种原始 auto_output 布局。）

   得到：
   - `auto_output/dataset_full_test/<object_slug 安全化>/<6位序号>/`
   - 每个 object（model）独占一级子目录，其下 000000、000001… 为该物体多条仿真样本，便于后续按物体做 train/test/val 划分。
   - `manifest.json`：`layout=by_model`、`models` 汇总、`samples` 含 `relative_path`、`index_in_model` 等。

   默认 `train_test` 布局不变：`auto_output/dataset_400/train|test/<编号>/`。
